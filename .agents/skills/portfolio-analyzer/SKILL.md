# 持仓组合分析

从券商持仓截图中提取持仓数据，结合实时行情和财务指标，对投资组合进行全面健康诊断。

## 工作流程

### 第一步：提取持仓数据

用户会提供 1-N 张券商持仓截图（JPG/PNG）。需要提取以下信息：

| 字段 | 说明 |
|------|------|
| 证券名称 | 股票/ETF/基金名称 |
| 数量 | 持有股数/份额 |
| 现价 | 当前价格 |
| 成本价 | 买入均价 |
| 市值 | 当前市值 |
| 持仓盈亏 | 盈亏金额 |
| 盈亏比例 | 盈亏百分比 |

**提取方式**：用 Read 工具直接读取用户提供的截图文件路径，模型自动解析图片内容。

用户消息中会包含图片文件路径（如 `/Users/xxx/Downloads/1.jpg`），直接读取即可。
不要用 Glob 或 ls 在项目目录中搜索——用户的图片不在项目目录中。

**如果无法读取图片**（模型不支持图片、消息中只有 `[Image]` 占位符没有路径等），
直接告知用户需要切换到支持图片理解的模型：

```
当前模型不支持读取图片。要分析持仓截图，请切换到支持图片理解的模型（如 Claude Sonnet/Opus、GPT-4o 等），
然后重新发送截图即可。

或者您也可以直接把持仓信息告诉我（股票名称、数量、成本价），我来分析。
```

> 重要：不要在没有路径的情况下反复用 Glob/ls 搜索图片，直接引导用户。

将提取结果整理为 Markdown 表格，**向用户确认数据是否正确**，再进行后续分析。

> **重要**：截图中可能包含 A 股、港股（标注"港元"）、ETF、场内基金等多种资产。
> 需要分别识别资产类型。港股市值需按当日汇率折算为人民币。

### 第二步：确认投资风格

如果用户没有主动说明自己的投资风格，通过交互确认。问用户：

```
在分析之前，想了解一下您的投资偏好，这样我可以从更贴合的角度分析：

1. **投资周期**：长线（1年以上） / 中线（1-6个月） / 短线（1个月内）
2. **风险偏好**：激进（追求高收益，能承受大回撤） / 稳健（适度收益，控制回撤） / 保守（保本优先，稳定收益）
3. **投资目标**：资产增值 / 稳定现金流（分红） / 打新底仓 / 其他

您也可以直接描述，比如"我是长线价值投资者"或"短线趋势交易"。
```

如果用户不回答或说"随便"/"都看看"，默认按「中线 + 稳健 + 资产增值」分析。

### 第三步：获取实时数据

对提取到的**个股**（非 ETF/基金），使用内置脚本批量获取数据：

```bash
python .agents/skills/portfolio-analyzer/scripts/batch_quote.py <代码1> <代码2> ... [--format table|json]
```

代码格式：`sh600519`、`sz000001`。一次性获取所有持仓的实时行情 + MA/MACD/RSI 等基础技术指标。

如果需要更深入的单只股票分析，再按需调用：
- **技术指标**（technical-indicator）：完整技术面分析
- **财务数据**（financial-data-analysis）：估值和基本面详细分析

> ETF 和场内基金只需获取实时行情，不需要财务分析。
> 港股目前不支持数据获取，仅基于截图数据分析。

### 第四步：组合诊断分析

基于提取的持仓数据和获取的实时数据，从以下维度进行分析：

#### 1. 持仓总览

- 总资产、股票市值、可用资金、仓位比例
- 总持仓盈亏（金额和百分比）
- 持仓数量统计（个股/ETF/港股各几只）

#### 2. 资产配置分析

| 分析维度 | 具体内容 |
|---------|---------|
| **资产类型** | A 股个股 vs ETF/基金 vs 港股 的市值占比 |
| **行业分布** | 是否过度集中于某一行业，行业间是否有对冲 |
| **市值分布** | 大盘蓝筹 vs 中小盘 的配比 |
| **风格分布** | 价值型 vs 成长型 vs 周期型 的配比 |

对配置给出评价：
- 集中度是否合理（单只占比超 20% 需提示风险）
- 行业是否过度集中（单行业超 30% 需提示）
- ETF 与个股的配比是否符合投资风格

#### 3. 盈亏诊断

| 分析维度 | 具体内容 |
|---------|---------|
| **盈亏分布** | 盈利持仓 vs 亏损持仓的数量和市值占比 |
| **深度套牢** | 亏损超 20% 的持仓，评估基本面是否恶化 |
| **浮盈管理** | 盈利超 50% 的持仓，分析是否需要止盈或减仓 |
| **成本合理性** | 结合当前估值，判断持仓成本在历史区间的位置 |

#### 4. 风险评估

| 风险类型 | 评估方法 |
|---------|---------|
| **集中度风险** | 单只持仓占比、前 3 大持仓占比 |
| **估值风险** | 高 PE/PB 持仓的占比 |
| **流动性风险** | 小市值、低成交量持仓 |
| **关联风险** | 多只股票属同一行业或上下游，可能同涨同跌 |
| **汇率风险** | 港股持仓的汇率暴露 |

#### 5. 基于投资风格的定制建议

根据第二步确认的投资风格，侧重不同分析角度：

**长线价值型**：
- 关注 ROE、分红率、护城河
- 亏损持仓是否值得"越跌越买"
- 估值是否仍在合理区间

**中线成长型**：
- 关注营收增速、利润增速
- 技术面趋势是否健康
- 是否有业绩拐点信号

**短线交易型**：
- 关注技术指标信号（MACD/RSI/KDJ）
- 量价配合情况
- 是否有需要止损的持仓

**保守稳健型**：
- 关注低波动 ETF 和蓝筹股占比
- 分红收益情况
- 整体回撤控制

### 第五步：输出分析报告

报告结构：

```markdown
# 持仓组合诊断报告

**分析日期**：YYYY-MM-DD
**投资风格**：XXX
**总资产**：XXX 元

## 一、持仓总览
（表格：持仓明细，含名称、数量、现价、成本、盈亏、占比）

## 二、资产配置
（饼图描述 + 评价）

## 三、盈亏诊断
（盈亏分布、套牢分析、止盈提示）

## 四、个股点评
（对每只重点持仓给出简短评价，含基本面 + 技术面信号）

## 五、风险提示
（按严重程度排列的风险点）

## 六、优化思路
（基于投资风格的配置调整方向，不给具体买卖建议）
```

## 注意事项

- 截图中的数据可能是盘中数据或收盘数据，分析前先判断时效性
- 截图可能不完整（需要滚动翻页），提示用户确认是否包含全部持仓
- 不给具体买卖建议（"应该买 X"），只指出信号和风险
- ETF 和基金不需要基本面分析，但需分析其跟踪指数的走势
- 港股持仓可能标注"港元"，需注意区分币种
- 总资产信息通常在第一张截图的顶部，注意提取
- 分析末尾加上：*以上分析基于公开数据和截图信息，仅供参考，不构成投资建议。*
- 完成分析后，主动问用户是否需要保存为投资笔记
