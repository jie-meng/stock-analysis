# 投资笔记管理

在 `notes/` 目录下管理投资研究笔记，帮助用户积累和回顾分析成果。

## 目录结构

```
notes/
├── stocks/           # 按股票归档
│   ├── 600519-贵州茅台/
│   │   ├── 2024-03-15-基本面分析.md
│   │   └── 2024-04-01-技术面跟踪.md
│   └── 000001-平安银行/
│       └── 2024-03-20-估值分析.md
├── market/           # 市场观察
│   └── 2024-03-15-白酒板块分析.md
└── strategy/         # 策略思考
    └── 我的选股标准.md
```

## 笔记模板

### 个股分析笔记

```markdown
# [股票名称] 分析笔记

**日期**：YYYY-MM-DD
**代码**：XXXXXX
**当前价格**：XX.XX
**分析类型**：基本面 / 技术面 / 综合

## 核心观点
（一句话结论）

## 分析过程
（详细分析内容）

## 风险提示
（需要关注的风险点）

## 后续计划
- [ ] 跟踪事项 1
- [ ] 跟踪事项 2
```

### 市场/行业笔记

```markdown
# [主题] 观察笔记

**日期**：YYYY-MM-DD

## 观察
（市场现象或行业动态）

## 判断
（个人分析和判断）

## 关注标的
（相关股票列表）
```

## 工作流程

### 创建笔记

当用户完成一次分析后说"记一下"或"保存"，按以下步骤：

1. 根据分析内容确定分类（stocks/market/strategy）
2. 使用模板创建笔记文件
3. 填入分析内容、当前数据、结论
4. 保存到对应目录

### 查阅笔记

当用户问"之前分析过 XXX 吗"或"看看笔记"：

1. 在 `notes/` 目录下搜索相关文件
2. 汇总呈现给用户
3. 对比当前数据与笔记中记录的数据

### 笔记汇总

当用户要求"汇总"或"回顾"：

1. 列出所有笔记的标题和日期
2. 按时间或按股票分组呈现
3. 标注待跟进事项

## 注意事项

- 笔记文件名使用 `YYYY-MM-DD-主题.md` 格式
- 每次分析后主动问用户是否需要保存笔记
- 笔记中包含当时的关键数据，方便日后回顾对比
