# A 股选股筛选器

内置多种预设策略 + 自定义条件，从全市场筛选符合条件的股票。

## 前置条件

```bash
pip install akshare pandas requests
```

## 使用方式

### 预设策略（推荐）

```bash
# 高分红策略：股息率高、连续分红、经营现金流健康
python .agents/skills/stock-screener/scripts/screener.py --strategy high-dividend

# 价值优选策略：低估值 + 高 ROE + 盈利增长 + 财务安全
python .agents/skills/stock-screener/scripts/screener.py --strategy value-quality

# 政策主题策略：十五五规划相关板块 + 基本面过滤
python .agents/skills/stock-screener/scripts/screener.py --strategy policy-theme [--theme 新能源]
```

### 自定义筛选

```bash
python .agents/skills/stock-screener/scripts/screener.py --custom \
  --pe-max 20 --pb-max 3 --roe-min 15 --div-yield-min 2 \
  --debt-ratio-max 50 --revenue-growth-min 5 \
  --sector 光伏概念
```

### 通用参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--strategy` | 预设策略：`high-dividend` / `value-quality` / `policy-theme` | 无 |
| `--theme` | 政策主题子方向（仅 policy-theme 策略），可选值见下表 | 全部主题 |
| `--custom` | 启用自定义模式 | 否 |
| `--top` | 输出前 N 只 | `30` |
| `--sort` | 排序字段：`roe` / `pe` / `div_yield` / `market_cap` | 策略默认排序 |
| `--exclude-st` | 排除 ST 股票 | 默认排除 |
| `--min-market-cap` | 最低市值（亿元） | `50` |

### 自定义条件参数

| 参数 | 说明 | 示例 |
|------|------|------|
| `--pe-max` | 市盈率上限 | `20` |
| `--pe-min` | 市盈率下限（排除亏损股） | `0` |
| `--pb-max` | 市净率上限 | `3` |
| `--roe-min` | ROE 下限（%） | `15` |
| `--div-yield-min` | 股息率下限（%） | `3` |
| `--debt-ratio-max` | 资产负债率上限（%） | `50` |
| `--revenue-growth-min` | 营收增速下限（%） | `10` |
| `--profit-growth-min` | 净利增速下限（%） | `5` |
| `--sector` | 限定概念板块（东方财富概念名称） | `光伏概念` |

## 预设策略详细说明

### high-dividend（高分红策略）

**投资逻辑**：寻找持续高分红、现金流健康的稳健型标的。

| 筛选条件 | 阈值 | 原因 |
|----------|------|------|
| 股息率 | ≥ 3% | 高于多数理财产品收益 |
| 连续分红年数 | ≥ 3 年 | 分红持续性 |
| PE | 0 < PE ≤ 30 | 排除亏损和泡沫 |
| 市值 | ≥ 50 亿 | 流动性保障 |

默认按股息率降序排序。

### value-quality（价值优选策略）

**投资逻辑**：低估值买入经营优秀的公司 — 好公司 + 便宜价格。

| 筛选条件 | 阈值 | 原因 |
|----------|------|------|
| PE | 0 < PE ≤ 25 | 估值合理 |
| PB | ≤ 5 | 资产未严重高估 |
| ROE | ≥ 12% | 资本回报能力 |
| 净利润增速 | ≥ 0% | 盈利未恶化 |
| 资产负债率 | ≤ 60% | 财务安全 |
| 市值 | ≥ 80 亿 | 有一定规模 |

默认按 ROE 降序排序。

### policy-theme（政策主题策略）

**投资逻辑**：在政策重点支持方向中，筛选基本面合格的标的。

第一步：通过概念板块框定政策相关股票池
第二步：用基本面条件二次过滤（PE/PB/市值）

可用主题方向（`--theme` 参数）：

| 主题 | 涵盖概念板块 |
|------|-------------|
| 新能源 | 光伏概念、风电概念、储能概念、新能源 |
| 半导体 | 半导体概念、国产芯片、第三代半导体、存储芯片 |
| 人工智能 | 人工智能、算力概念、AI芯片 |
| 生物医药 | 生物疫苗、合成生物 |
| 高端制造 | 机器人概念、人形机器人、新材料 |
| 绿色经济 | 碳中和（需确认板块名称可用性） |

默认按市值降序排序。

## 输出格式

Markdown 表格 + 筛选统计摘要。示例：

```
# 选股结果：高分红策略

全市场 5200 只 → 粗筛 320 只 → 精筛 45 只 → 展示 Top 30

| 排名 | 代码 | 名称 | 股息率 | PE | PB | ROE | 市值(亿) |
|------|------|------|--------|-----|-----|------|----------|
| 1 | 601288 | 农业银行 | 6.2% | 5.1 | 0.5 | 11.2 | 12800 |
| ... | ... | ... | ... | ... | ... | ... | ... |
```

## 筛选机制说明

采用两阶段筛选以平衡速度和精度：

1. **粗筛**（快，秒级）：调用全市场实时接口，按 PE/PB/市值/ST 等条件批量过滤
2. **精筛**（较慢，取决于粗筛结果量）：对粗筛结果逐个获取详细财务/分红数据

粗筛阶段如果结果超过 200 只，会收紧条件自动缩小范围，避免精筛阶段过慢。

## 注意事项

- 全市场数据获取需要 1-3 分钟，取决于网络和 akshare 接口响应速度
- 筛选结果基于最新公开数据，财务数据可能有 1-2 个季度延迟
- 政策主题板块映射需要人工维护更新（`sectors.json`）
- 筛选结果是进一步分析的起点，建议对感兴趣的个股使用 financial-data-analysis 和 technical-indicator 技能深入分析
